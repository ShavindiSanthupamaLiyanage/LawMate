# Contributing to LawMate

## Branch Strategy
- `main` - Production code (NO DIRECT PUSHES)
- `development` - Testing code (NO DIRECT PUSHES)
- `feature/*` - Feature branches (YOUR WORK HERE)

## Workflow Rules
1. NEVER push directly to `main` or `development`
2. Always create feature branches
3. Always create Pull Requests
4. Require at least 1 review before merging
5. Resolve all comments before merging
6. Test your code before creating PR

## Branch Naming
- Features: `feature/name-description`
- Bugs: `bugfix/name-description`
- Hotfix: `hotfix/description`

## Commit Message Format
- `Add: description` - New feature
- `Fix: description` - Bug fix
- `Update: description` - Update existing code
- `Refactor: description` - Code refactoring
- `Docs: description` - Documentation

## Code Review Checklist
- [ ] Code works as intended
- [ ] No console errors
- [ ] Follows code style
- [ ] Commented where necessary
- [ ] No security issues
